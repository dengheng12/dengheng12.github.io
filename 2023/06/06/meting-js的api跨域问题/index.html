meting-js的api跨域问题

发表于2023-06-06|更新于2025-07-02|分类1
|阅读量:

1、起因

&nbsp; 本来快快乐乐的吃着火锅，唱着歌，突然就被。。。哈哈哈，又到了可以闲下来写博客的时间，本来也没有什么的，就想听听歌，但是，突然我的歌单不见了。。。

2、分析

&nbsp; 刚开始我以为是网络原因，刷新试了下，还是不行，我又试试重新启动hexo服务 hexo s 但是还是不行，百度了下也没有个所以然，无奈之下，我打开了控制台，发现是meting-js的api报跨域了

3、开始解决

&nbsp; 问题是找到了，怎么解决呢，我先百度了一翻，还是没有找到对应的结果，无奈去github上搜搜，找到meting-js的github地址，粗略的翻阅了下 readme文件，没有找到我想要的答案

4、尝试找答案

&nbsp; 找到issue碰碰运气，没想到还真有，就是这个，浏览器上试了下，发现可以，就在我满心欢喜，感觉胜利的曙光尽在眼前的时候。。。

5、初始解决

&nbsp; 我想到的是能直接替换meting-js的请求路径不就完事了么，我直接找到node_modules ，找到meting-js的文件，直接替换，然而，我本地重新运行服务的时候，发现并没有什么卵用，我再次打开了控制台，发现meting-js居然是引用的cdn资源，我。。。，百度了下，找到配置文件 #

配置成local，然后运行 npm i hexo-butterfly-extjs 重新运行服务 发现meting-js的请求路径变了

/pluginsSrc/butterfly-extsrc/metingjs/dist/Meting.min.js

这。。。，到哪里找呢，我灵机一动，到node_modules找到butterfly主题下的/butterfly-extsrc/metingjs/dist/Meting.min.js 修改打印信息，重新运行，发现打印的信息变了，就是这样

6、暂时解决

&nbsp; 接下来就是替换meting-js的api接口请求路径 替换成

https://meting.yany.ml/api  但是。。。 tencent的歌曲链接全部报404 无奈替换成网易的server，发现居然可以，皆大欢喜吧！！！

7、总结一下

&nbsp; 最好的解决方案应该是有一个直接替换的meting-js的api 选项，但是目前还没找到，暂时先这样吧
